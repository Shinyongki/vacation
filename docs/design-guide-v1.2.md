> 작성일: 2026.04.12

> 디자인 방향: 공공기관(A) 기반 + SaaS(B) 마감 + 복지기관(C) 가이드 카드

## 1. 디자인 철학

### 1.1 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 공공기관의 신뢰감 | 네이비 블루 상단 헤더, 아코디언 사이드바, 테이블 기반 데이터 표시. 기존 공공 시스템에 익숙한 사용자가 자연스럽게 받아들이는 구조 |
| 현대적 마감 | 0.5px 보더, 8px 라운드 코너, pill 배지, 카드형 요약 영역. 2010년대 공공 시스템이 아닌 현대적 느낌 |
| 맥락적 안내 | 사용자가 할 일이 있을 때만 가이드 카드(노란 배너) 표시. 모든 화면에 넣지 않음 |

### 1.2 하지 않는 것

- 이모지를 메뉴나 레이블에 사용하지 않음
- "안녕하세요, OO님" 같은 인사말 없음 (복지기관 톤 제외)
- 도넛 차트 같은 과도한 시각화 없음 (프로그레스 바로 충분)
- 그라디언트, 그림자, 글로우 효과 없음
- 아이콘만 있는 슬림 사이드바 없음 (텍스트 메뉴 유지)

## 4. 레이아웃

### 4.1 전체 구조

```
┌─────────────────────────────────────────────────────────┐
│  상단 헤더 (44px, 네이비 #1B3A5C)                         │
│  [로고+시스템명]                    [알림] [사용자] [로그아웃] │
├─────────────────────────────────────────────────────────┤
│  경로 바 (28px, #EEF1F5)    홈 > 대시보드                  │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│ 사이드바  │  콘텐츠 영역                                   │
│ (192px)  │  (나머지, padding 20px 24px)                   │
│ #F6F7F9  │  #F9FAFB                                     │
│          │                                              │
│          │                                              │
│          │                                              │
│          │                                              │
│          │                                              │
├──────────┴──────────────────────────────────────────────┤
```

### 4.2 상단 헤더 (44px)

- 배경: `#1B3A5C`
- 좌측: 로고(26×26px, 라운드 6px, 배경 `#2E6DA4`) + 시스템명 (#fff, 14px, 500)
- 우측: 알림 벨 + 미읽은 수 배지(빨강), 사용자명+직위 (12px, `#C8D8E8`), 로그아웃 (11px, `#7A9AB8`)

### 4.3 경로 바 (breadcrumb, 28px)

- 배경: `#EEF1F5`
- 하단 보더: 0.5px solid `#D8DDE5`
- 텍스트: 11px, `#5A6E82`
- 구분자: `>`

### 4.4 사이드바 (192px) — 네이비 다크 스타일

- 배경: **`#1B3A5C`** (상단 헤더와 동일 계열)
- border-right: 없음
- box-shadow: 없음

**카테고리 그룹 제목** (나의 공간, 휴가 관리, 현황, 정보 등):
- font-size: 13px, font-weight: 500, color: `#93B3D1`
- letter-spacing: 0.3px
- padding: 10px 16px
- margin-top: 20px (첫 번째 그룹 제외)
- border-bottom: 1px solid rgba(255,255,255,0.08)

**하위 메뉴 (비활성)**:
- font-size: 15px, font-weight: 400, color: `#C8D8E8`
- padding: 10px 16px 10px 24px (좌측 들여쓰기)
- hover: background rgba(255,255,255,0.06)

**하위 메뉴 (활성)**:
- font-size: 15px, font-weight: 500, color: `#FFFFFF`
- background: rgba(255,255,255,0.12)
- border-left: 3px solid `#60A5FA`
- padding: 10px 16px 10px 21px

**아코디언 화살표**: color `#7A9AB8`, hover `#93B3D1`

**하단 사용자 정보**:
- border-top: 1px solid rgba(255,255,255,0.1)
- 부서명: 12px, `#7A9AB8`
- 이름·직위: 14px 500, `#FFFFFF`
- 팀장·원장: 부재중 토글 (AbsenceToggle) 표시

### 4.5 반응형

- 최소 너비: 1024px (시설 PC 기준)
- 사이드바: 고정 192px
- 콘텐츠: 나머지 영역
- 모바일 대응 불필요 (PC 전용)

## 9. 접근성

- 최소 폰트: 11px (배지, 힌트)
- 일반 텍스트: 12~13px
- 색상 대비: WCAG AA 기준 충족 (배경 대비 4.5:1 이상)
- 포커스 링: 보더 2px solid #1B5E9E (키보드 네비게이션)
- 비활성 요소: opacity 0.5 + cursor not-allowed
- 상태 표시: 색상 + 텍스트 병행 (색맹 대응)
