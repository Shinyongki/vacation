## 1. 시스템 개요

- **시스템명**: 휴가관리 시스템
- **사용 환경**: 시설 내부 PC에서 .exe 실행 → 같은 네트워크 내 브라우저 접속
- **사용자 수**: 시설 당 약 10~50명
- **인터넷 의존**: 없음 (로컬 네트워크 전용)

#### 3.3.1 신청 폼 (L-03) 상세 정보 요소

**1단계 — 유형 선택**
- 대분류 7가지: 연차, 병가, 경조사, 공가, 출산휴가, 생리휴가, 포상휴가
- 연차 선택 시 하위 옵션: 종일 / 오전반차 / 오후반차 / 시간연차
- 경조사 선택 시: 세부 유형 드롭다운 + 규정 일수 즉시 표시 (녹색 박스)

**2단계 — 날짜·시간 선택**
- 종일: 달력 위젯 범위 선택 (시작일~종료일)
- 반차: 달력 위젯 단일 날짜 선택 + 오전/오후 자동 표시
- 시간연차: 단일 날짜 + 시작 시각~종료 시각 (30분 단위 드롭다운), 점심시간 12:00~13:00 자동 제외

**3단계 — 확인 정보**
- 차감 일수 실시간 계산 표시
- 잔여 일수 프로그레스 바 (신청 전 → 신청 후)
- 해당일 팀 출근 현황 표시
- "긴급" 체크박스 (체크 시 긴급 사유 입력 필수)
- 사유 입력란 (병가·경조사·사후신청: 필수 / 그 외: 선택)
- 첨부파일 (병가 6일 초과 시 진단서 필수 안내)
- 사후 신청 여부 자동 판별 → 사유 카테고리 6가지 중 선택 필수 + 증빙 서류 첨부 안내
- **열람 범위 설정 (팀별 체크박스)**
  - 부서 목록 표시 (부서명, 인원수)
  - 내 팀: 기본 체크 (해제 가능)
  - 다른 팀: 기본 미체크
  - 안내: "승인 후 선택한 팀의 캘린더에 이름·유형·기간이 표시됩니다. 사유는 공개되지 않습니다."

**재기안/재상신 진입 시 폼 동작:**
- 재상신 (반려 건): 이전 신청 데이터로 각 필드 미리 채움 + 상단에 반려 사유 노란 안내 박스 표시
- 재기안 (회수 건): 이전 신청 데이터로 각 필드 미리 채움
- 열람 범위도 이전 설정 그대로 복사
- 수정 후 "상신" 버튼으로 제출 → 새 건 생성 (pending)

| L-04 | 취소 확인 | 신청 상세 화면의 "취소" 버튼 | 취소 완료 또는 닫기 |
| **L-05** | **회수 확인** | **신청 상세 화면의 "회수" 버튼 (pending 건, 기안자 본인만)** | **회수 완료 또는 닫기** |
| A-03 | 반려 사유 | 승인 대기 목록의 "반려" 버튼 | 반려 완료 또는 닫기 |
| A-04 | 일괄 승인 | 원장 승인 대기 목록의 "일괄 승인" 버튼 | 승인 완료 또는 닫기 |
| C-04 | 비밀번호 변경 | 사이드바 하단 계정 메뉴 | 변경 완료 또는 닫기 |
| S-08 | 엑셀 업로드 | 관리자 설정 데이터관리 탭 | 등록 완료 또는 닫기 |

## 5. 화면 간 네비게이션 흐름

### 5.1 진입 흐름
```
로그인 (C-01)
  ├─ 직원 → 직원 대시보드 (D-01)
  ├─ 팀장 → 팀장 대시보드 (D-02)
  ├─ 원장 → 원장 대시보드 (D-03)
  ├─ HR관리자 → HR 대시보드 (D-04)
  └─ 재단담당자 → 재단 대시보드 (D-05)
```

### 5.2 주요 사용자 흐름

**휴가 신청 흐름**
```
대시보드 (D-01~04)
  → "휴가 신청" 버튼
    → 신청 폼 모달 (L-03)
      → 유형 선택 → 날짜/시간 선택 → 확인 정보 + 열람 범위 설정 → 상신
        → 내 휴가 목록 (L-01) 으로 이동 (신청 완료 상태)
          → 신청 상세 (L-02) 에서 결재 진행 추적
```

**승인 흐름**
```
알림 수신 또는 대시보드 (D-02, D-03)
  → 승인 대기 목록 (A-01)
    → 개별 건 클릭 → 신청 상세 (L-02) 열람
      → "승인" → 다음 단계로 전달 (또는 최종 승인 완료)
      → "반려" → 반려 사유 모달 (A-03) → 신청자에게 알림
    → (원장) "일괄 승인" → 일괄 승인 모달 (A-04)
```

**회수 흐름 (v1.1 신규)**
```
내 휴가 목록 (L-01)
  → pending 건 선택 → 신청 상세 (L-02)
    → "회수" 버튼 → 회수 확인 모달 (L-05)
      → 사유 입력(선택) → "확인"
        → 건 상태: recalled, 결재 초기화
        → 결재자에게 "회수됨" 알림
          → 내 휴가 목록에서 "재기안" 버튼 표시
```

**재기안 흐름 (v1.1 신규)**
```
내 휴가 목록 (L-01)
  → recalled 건의 "재기안" 버튼 클릭
    → 신청 폼 모달 (L-03) — 이전 데이터 미리 채움
      → 수정 → "상신"
        → 새 건 생성 (pending, parent_request_id = 원본 ID)
```

**재상신 흐름 (v1.1 신규)**
```
내 휴가 목록 (L-01)
  → rejected 건의 "재상신" 버튼 클릭
    → 신청 폼 모달 (L-03) — 이전 데이터 + 반려 사유 표시
      → 반려 사유 참고하여 수정 → "상신"
        → 새 건 생성 (pending, parent_request_id = 원본 ID)
```

**직원 등록 흐름 (HR)**
```
관리자 설정 → 직원 관리 탭 (S-01)
  → "신규 등록" 버튼
    → 사번, 이름, 부서, 직위, 역할, 입사일, 생년월일, 성별, 연락처, 고용형태 입력
      → 저장 → employees 테이블 등록
        → 해당 사번으로 즉시 로그인 가능
        → 연차 자동 계산 시작 (leave_balances 생성)
```

**퇴사 처리 흐름 (HR)**
```
관리자 설정 → 직원 관리 탭 (S-01)
  → 직원 선택 → "퇴사 처리" 버튼
    → 확인 → status를 inactive로 변경, resignation_date 자동 기록
      → 로그인 차단
      → 결재 라인에서 제외 (대결자 자동 전환)
      → 데이터 보존 (이력 열람 가능)
```

**데이터 이전 흐름 (HR)**
```
관리자 설정 → 데이터관리
  → "엑셀 업로드" → 엑셀 업로드 모달 (S-08)
    → 파일 선택 → 필드 매핑 → 미리보기 → 확인
      → 일괄 등록 완료
```

**데이터 검증 흐름 (재단담당자)**
```
데이터 검증 메뉴 (재단담당자 사이드바)
  ├─ 보고서 검증 (X-01)
  │    → 검증코드 입력 → "검증" 버튼
  │      → 원본 확인 (일치) 또는 데이터 변경 감지 (불일치) 표시
  ├─ 내보내기 이력 (X-02)
  │    → 기간 필터로 조회 → 누가, 언제, 몇 번 내보냈는지 확인
  │      → 동일 기간 반복 출력 시 경고 표시
  └─ 조정 이력 조회 (X-03)
       → 기간 필터로 조회 → 수동 조정 내역 전체 확인
         → 보고 직전 대량 조정 시 경고 표시
```

**보고서 내보내기 흐름 (원장·HR·재단)**
```
보고서 메뉴 → 기간·유형 선택 → "내보내기" 버튼
  → 시스템이 동시에 생성:
    ├─ 엑셀 파일 (작업용)
    └─ PDF 파일 (검증코드 삽입된 원본 증빙)
  → zip 파일로 다운로드
  → export_logs에 이력 자동 기록
```

### 5.3 알림 → 화면 연결

| 알림 유형 | 클릭 시 이동 |
|-----------|-------------|
| 새 결재 요청 | 승인 대기 목록 (A-01) |
| 승인 완료 | 신청 상세 (L-02) |
| 반려 통보 | 신청 상세 (L-02) |
| 대결 요청 | 승인 대기 목록 (A-01) - 대결 건 필터 |
| 취소 완료 | 내 휴가 목록 (L-01) |
| 긴급 건 알림 | 승인 대기 목록 (A-01) - 긴급 건 강조 |
| 부재 자동 전환 | 승인 대기 목록 (A-01) - 대결 건 |
| 미처리 경고 | 승인 대기 목록 (A-01) - 해당 건 강조 |
| **회수됨** | **내 휴가 목록 (L-01) — 해당 건이 목록에서 사라짐 확인** |

## 6. 정보 계층 구조

### 6.1 데이터 계층

```
시설 (시스템 단위)
  └─ 부서 (departments)
       └─ 직원 (employees)
            ├─ 휴가 잔여일수 (leave_balances) — 연도별
            │    └─ 잔여일수 조정 이력 (balance_adjustments)
            ├─ 휴가 신청 (leave_requests)
            │    ├─ 결재 단계 (approval_steps) — 실제 이력
            │    ├─ 열람 범위 (leave_request_visibility) — 팀별 공개 설정
            │    └─ 재기안/재상신 연결 (parent_request_id)
            ├─ 대결자 지정 (delegates)
            └─ 내보내기 이력 (export_logs) — 검증코드 포함
```

### 6.2 설정 계층

```
시스템 설정 (system_settings)
  ├─ 휴가 유형 (leave_types)
  │    └─ 경조사 세부유형 (condolence_subtypes)
  ├─ 결재 라인 템플릿 (approval_flows)
  │    └─ 결재 단계 정의 (approval_flow_steps)
  ├─ 유형↔결재라인 연결 (leave_type_flow_mapping)
  ├─ 공휴일·자체휴일 (holidays)
  └─ 알림 (notifications)
```
