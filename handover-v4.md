## 지금 어떤 작업을 하고 있나

재단법인 경상남도사회서비스원 산하 시설에 납품할 **휴가관리 프로그램**을 기획·설계 중이다.
개발은 Claude Code에서 할 예정이고, **지금은 Claude.ai에서 기획·설계·목업·작업지시서 작업을 진행하고 있다.**
Claude AI가 전체 방향 설정·기획·설계·의사결정을 담당하고, Claude Code 작업 진행 사항을 Claude AI와 논의하며 협업하는 구조다.

**현재 단계**: 기획·설계 완료 → Claude Code 작업 지시서 완성 → **개발 착수 가능 상태**

## 기술 및 납품 방식

- React + Node.js + Express + SQLite
- **Node.js SEA (Single Executable Applications)** 로 .exe 단일 실행파일 패키징
- 같은 네트워크 내 브라우저로 접속하는 방식 (시설 PC에서 .exe 실행 → 내부 웹 서버)
- 알림: 이메일 없음, **앱 내 알림만** (인터넷 불필요)
- 재단 담당자: 엑셀+PDF 동시 내보내기, 추후 클라우드 동기화 추가 가능 (보류)
- 유지보수: 개발자(나)가 직접 계속 관리

## v3 → v4 주요 변경사항 (이번 세션)

### 디자인 가이드 확정
- 3가지 디자인 톤 비교 (공공기관 / SaaS / 복지기관) → **A(공공기관) + B(SaaS 마감)** 혼합 확정
- 상단 헤더바(네이비) + 아코디언 사이드바 + 테이블 기반 + 0.5px 보더 + 8px 라운드 + pill 배지
- 가이드 카드 (노란 배너, 할 일 있을 때만)
- 결재 진행 도트 (테이블 인라인) + 결재 상세 모달 (타임라인 UI)

### 회수 + 재기안 + 재상신 기능 (신규)
- **회수(recalled)**: 기안자가 pending 건을 직접 끌어옴. 결재 초기화. "재기안"으로 수정 후 다시 올릴 수 있음
- **재기안**: 회수 건에서 이전 데이터 복사 → 수정 → 새 건 생성 (parent_request_id로 원본 연결)
- **재상신**: 반려 건에서 이전 데이터 + 반려 사유 표시 → 수정 → 새 건 생성
- **취소 vs 회수 구분**: 취소=건 종료(재기안 불가), 회수=수정 목적(재기안 가능)
- 상태값: 4개→6개 (draft 예약, recalled 추가)

### 열람 범위 설정 (신규)
- 기안자가 신청 폼에서 **팀별 체크박스**로 공개 범위 선택
- 내 팀 기본 체크 (해제 가능), 다른 팀 체크 가능
- 승인 후 해당 팀 캘린더에만 이름+유형+기간 표시 (사유 비공개)
- 팀장·원장·HR은 권한으로 전체 열람 가능 (열람 범위와 별개)
- leave_request_visibility 테이블 신규

### 직원 정보 필드 확장 (신규)
- 등록 필드: 7개→**11개** (성별, 연락처, 고용형태, 퇴사일 추가)
- **성별**: 생리휴가·출산휴가 자격 검증에 사용. 남성은 해당 유형 비활성화
- **고용형태**: 정규직/계약직. HR관리자가 고용형태별 사용 가능 휴가 유형을 체크박스로 설정
- **퇴사일**: 퇴사 처리 시 시스템이 자동 기록
- employment_type_leave_mapping 테이블 신규

### 브라우저 알림 추가 (신규)
- 앱 내 알림 + **Web Notification API** 병행
- 시스템 탭이 열려있으면 다른 프로그램 사용 중에도 바탕화면 팝업 표시
- 30초마다 폴링으로 새 알림 감지
- 인터넷 불필요 (로컬 네트워크에서 동작)

### Claude Code 오케스트라 병렬 코딩 구조 확정
- 작업 지시서 v2.0 §9에 상세 기술
- **Phase 0(단독) → Phase 1(4병렬) → Phase 2(3병렬) → Phase 3(2병렬) → Phase 4(통합)**
- 트랙별 파일 소유권 매트릭스로 충돌 방지
- 에이전트별 복사-붙여넣기 프롬프트 포함
- Phase Gate 체크리스트로 단계 간 품질 확인

### 임시보관(draft) 스키마 예약
- status에 'draft' 추가했지만 현재 미구현
- 추후 "작성 중 저장" 기능 추가 시 바로 사용 가능

## 작업 스타일

- 목업은 Claude visualize 위젯으로 인터랙티브하게 만들어왔음
- 설명은 표나 다이어그램으로 시각화해서 진행
- 개발 전 단계라 "왜 이렇게 만드는지" 이유를 충분히 논의하며 진행
- 질문할 게 있을 때는 선택지 버튼으로 물어봐 주면 좋음
- "가능한 것 다 만들고 빼기" 접근 — 확정된 요구사항 기준으로 넓게 설계하고, 답변 오면 조정
- 문서 정합성 검증을 꼼꼼히 진행 — 문서 간 수치·참조 교차 검증
- 디자인 결정은 목업으로 비교 후 확정 — 이번 세션에서 3종 비교 후 혼합안 확정
